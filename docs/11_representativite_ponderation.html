11  Représentativité et pondération

WarningTravail de vérification en cours

Les résultats présentés ci-dessous sont provisoires et susceptibles de corrections.

Ce chapitre examine dans quelle mesure l’échantillon 2025 des observatoires ruraux est représentatif des populations des zones d’étude, et propose des stratégies de pondération pour corriger les biais d’échantillonnage. La démarche s’articule en trois volets : (i) caractériser le plan de sondage et calculer les poids de base, (ii) confronter l’échantillon aux données du Recensement Général de la Population et de l’Habitation (RGPH-3) de 2018, et (iii) discuter les options de calage et d’estimation pour petits domaines.

L’analyse repose sur la théorie de l’échantillonnage assistée par modèle (Särndal, Swensson, and Wretman 2003) et sur les méthodes de calibration (Deville and Särndal 1992). Les calculs sont réalisés avec le package survey pour R (Lumley 2010, 2004).

11.1 Plan de sondage

11.1.1 Structure de l’échantillon

Les observatoires ruraux reposent sur un plan de sondage à deux degrés dans lequel les hameaux constituent les unités primaires de sondage (UPS) et les ménages les unités secondaires. Au sein de chaque UPS, un tirage systématique est opéré à partir de la liste de dénombrement.

En 2025, après dix ans d’interruption, l’échantillon a été reconstitué via un nouveau dénombrement exhaustif suivi d’un tirage de ménages. Une partie des ménages tirés sont d’anciens ménages du panel, tandis que d’autres sont entièrement nouveaux. Les probabilités d’inclusion varient donc d’un hameau à l’autre, ce qui impose l’utilisation de pondérations pour obtenir des estimations non biaisées (Kish 1965; Lohr 2022).

11.1.2 Taux de sondage par hameau

Le taux de sondage varie sensiblement d’un hameau à l’autre. Pour chaque ménage, le poids de base (ou poids de design) est l’inverse du taux de sondage de son hameau :

\[w_i = \frac{N_h}{n_h}\]

où \(N_h\) est le nombre de ménages dénombrés dans le hameau \(h\) et \(n_h\) le nombre de ménages enquêtés (Kish 1965).

Code 
# Ménages enquêtés par hameau
enquetes_ham <- res_deb |>
  count(Observatory, j4, name = "n_enquetes")

# Ménages dénombrés par hameau
denom_ham <- enum_all |>
  group_by(Observatory, Hameau) |>
  summarise(
    n_denombrés = sum(`Nbr ménage D`, na.rm = TRUE),
    .groups = "drop"
  )

# Clé normalisée pour l'appariement
normalize_name <- function(x) {
  x |>
    str_to_lower() |>
    stringi::stri_trans_general("Latin-ASCII") |>
    str_squish()
}

enquetes_ham <- enquetes_ham |>
  mutate(key = normalize_name(j4))

denom_ham <- denom_ham |>
  mutate(key = normalize_name(Hameau))

sampling <- denom_ham |>
  left_join(enquetes_ham, by = c("Observatory", "key")) |>
  mutate(
    n_enquetes = replace_na(n_enquetes, 0),
    taux_sondage = ifelse(n_denombrés > 0,
                          round(n_enquetes / n_denombrés * 100, 1), NA_real_),
    poids_base = ifelse(n_enquetes > 0, n_denombrés / n_enquetes, NA_real_)
  )

sampling |>
  select(Observatory, Hameau, n_denombrés, n_enquetes, taux_sondage, poids_base) |>

11.2 Estimations pondérées

Nous construisons un objet svydesign en spécifiant les hameaux comme strates. Dans la mesure où les hameaux sont exhaustivement couverts (il ne s’agit pas d’un tirage d’UPS parmi un ensemble plus large), chaque hameau est traité comme une strate avec un tirage à un seul degré.

Code 
# Rattacher les poids aux données ménages
deb_w <- res_deb |>
  mutate(key = normalize_name(j4)) |>
  left_join(
    sampling |> select(Observatory, key, poids_base, n_denombrés),
    by = c("Observatory", "key")
  ) |>
  mutate(poids_base = replace_na(poids_base, 1))

# Objet svydesign
design_or <- svydesign(
  ids = ~1,
  strata = ~j4,
  weights = ~poids_base,
  data = deb_w,
  nest = TRUE
)

11.2.1 Comparaison pondérée / non pondérée

L’écart entre les estimations pondérées et non pondérées indique dans quelle mesure les taux de sondage inégaux affectent les statistiques descriptives. Un écart faible suggère un plan approximativement auto-pondéré ; un écart important justifie l’usage systématique des poids.

Code 
# Taille du ménage
hh_size <- res_m_a |>
  count(j0, j5, Observatory, name = "taille")

deb_w2 <- deb_w |>
  left_join(hh_size |> select(j0, j5, taille), by = c("j0", "j5"))

design_or2 <- svydesign(
  ids = ~1,
  strata = ~j4,
  weights = ~poids_base,
  data = deb_w2,
  nest = TRUE
)

# Non pondéré
np <- deb_w2 |>
  summarise(
    Moyenne = mean(taille, na.rm = TRUE),
    ET = sd(taille, na.rm = TRUE),
    .by = Observatory
  ) |>
  mutate(Type = "Non pondéré")

# Pondéré
pond_al <- svymean(~taille,
                   subset(design_or2, Observatory == "Alaotra"),
                   na.rm = TRUE)
pond_mr <- svymean(~taille,
                   subset(design_or2, Observatory == "Marovoay"),
                   na.rm = TRUE)

p <- tibble(
  Observatory = c("Alaotra", "Marovoay"),
  Moyenne = c(coef(pond_al), coef(pond_mr)),
  ET = c(SE(pond_al), SE(pond_mr)),
  Type = "Pondéré"
)

bind_rows(np, p) |>
  pivot_wider(names_from = Type, values_from = c(Moyenne, ET),
              names_glue = "{Type}_{.value}") |>
  gt() |>
  cols_label(
    Observatory = "Observatoire",
    `Non pondéré_Moyenne` = "Moyenne",
    `Non pondéré_ET` = "ET / ES",
    `Pondéré_Moyenne` = "Moyenne",
    `Pondéré_ET` = "ET / ES"
  ) |>
  tab_spanner(label = "Non pondéré", columns = starts_with("Non")) |>
  tab_spanner(label = "Pondéré", columns = starts_with("Pond")) |>
  fmt_number(decimals = 2) |>
  tab_source_note(
    source_note = md("ET = écart-type (non pondéré) ; ES = erreur standard (pondéré)")
  ) |>
  style_table()

11.3 Comparaison avec le RGPH 2018

Pour évaluer la représentativité externe de l’échantillon, nous confrontons certaines caractéristiques aux données du 3e Recensement Général de la Population et de l’Habitation (RGPH-3) de 2018 (INSTAT 2020). Le RGPH-3 fournit un échantillon de 10 % des ménages et résidents au niveau national.

NotePérimètre et limites de la comparaison

Trois réserves sont nécessaires :

  1. Écart temporel : sept années séparent le RGPH (2018) de l’enquête (2025). Les structures démographiques ont pu évoluer, notamment du fait des migrations, de la transition démographique et des chocs climatiques récurrents.

  2. Maille géographique : le RGPH-3 ne descend pas au niveau du fokontany. Nous comparons au niveau des districts d’Amparafaravola (code 333) et de Marovoay (code 414), en ne retenant que le milieu rural (MILIEU = 2). Les observatoires couvrent des hameaux spécifiques au sein de ces districts ; les profils démographiques des autres communes du même district peuvent différer.

  3. Nature de la source : le RGPH est un recensement de droit (population résidente habituelle) alors que l’enquête OR interroge les ménages présents au moment de l’enquête.

Ces comparaisons doivent donc être lues comme des ordres de grandeur, non comme des tests formels de représentativité.

11.4 Discussion méthodologique

11.4.1 Poids de sondage de base

Le poids de base \(w_i = N_h / n_h\) cale déjà l’effectif total de ménages par hameau sur le dénombrement. Pour les indicateurs calculés sur l’ensemble de l’observatoire, ces poids corrigent le biais dû aux taux de sondage inégaux entre hameaux. Si l’écart entre estimations pondérées et non pondérées est faible, le plan est approximativement auto-pondéré et la correction a un impact limité.

11.4.2 Calage sur marges (calibration)

Au-delà des poids de base, un calage sur marges permettrait d’ajuster les poids pour que la distribution pondérée de certaines variables auxiliaires (sexe, âge, taille du ménage) corresponde exactement à des totaux connus dans la population cible (Deville and Särndal 1992; Haziza and Beaumont 2017). Les sources de calage envisageables sont :

  1. Dénombrement 2025 : nombre total de ménages et de toits par hameau — déjà exploité par les poids de base.

  2. RGPH 2018 : si les données sont disponibles au niveau district, un calage sur la structure par sexe, âge ou taille du ménage corrigerait les biais de couverture. L’écart temporel (sept ans) constitue toutefois une limite importante : une population cible datée de 2018 ne reflète plus nécessairement la structure de 2025.

  3. Projections démographiques INSTAT : des projections communales post-RGPH pourraient servir de marges actualisées, si elles sont rendues disponibles.

La méthode de Deville and Särndal (1992), implémentée dans survey::calibrate(), minimise la distance entre poids initiaux et poids finaux sous les contraintes de marges (calage linéaire, raking ou borné).

11.4.3 Estimation pour petits domaines

L’effectif de l’échantillon (environ 500 ménages par observatoire) impose des limites à la finesse de la désagrégation. Les estimations par hameau ou par sous-groupe (sexe × activité × hameau) comportent des intervalles de confiance larges et des risques de non-représentativité (Rao and Molina 2015).

Deux familles de méthodes permettent de produire des estimations fiables pour de petits domaines géographiques (Rao and Molina 2015; Battese, Harter, and Fuller 1988) :

  • Estimateurs de type EBLUP (Battese-Harter-Fuller) : ils combinent l’information directe de l’échantillon avec un modèle auxiliaire de régression exploitant des covariables connues au niveau de l’ensemble de la population (par exemple issues du RGPH). L’idée est « d’emprunter de la force » aux domaines voisins ou aux relations statistiques observées sur l’ensemble des données.

  • Estimateurs de type Fay-Herriot : variante adaptée lorsque seules les estimations directes agrégées par domaine (et non les micro-données) sont disponibles, couplées à des covariables auxiliaires au même niveau d’agrégation.

Dans le contexte des observatoires ruraux, une estimation pour petits domaines permettrait de produire des indicateurs au niveau des communes ou des fokontany en exploitant les structures apprises sur l’ensemble de l’échantillon et les marges du RGPH. Cette approche présuppose toutefois un modèle reliant correctement les variables d’intérêt aux covariables auxiliaires — hypothèse dont la validité est discutable en l’absence de mise à jour du recensement.

11.4.4 Post-stratification

La post-stratification est une forme simplifiée de calage. Elle consiste à partitionner l’échantillon en strates définies a posteriori (par exemple selon le sexe ou la classe d’âge du chef de ménage), puis à repondérer chaque strate de sorte que sa contribution au total pondéré corresponde à la proportion connue dans la population (Särndal, Swensson, and Wretman 2003, chap. 7). Si les strates de post-stratification sont corrélées à la variable d’intérêt, la variance de l’estimateur est réduite.

Avec deux observatoires et quelques variables de stratification, le nombre de cellules de post-stratification reste modeste, ce qui rend la méthode applicable. Son intérêt dépend néanmoins de la qualité et de l’actualité des marges disponibles.

TipRecommandations

  1. Poids de base : les statistiques descriptives de ce rapport ne sont pas pondérées, les taux de sondage étant relativement homogènes au sein de chaque observatoire. Une analyse de sensibilité systématique (comparaison pondéré / non pondéré) est recommandée pour les indicateurs publiés.

  2. Calage : un calage sur la structure par sexe et âge du RGPH 2018, en retenant le milieu rural des districts concernés, améliorerait la représentativité externe. L’écart temporel (2018–2025) est une limite à garder à l’esprit.

  3. Estimation pour petits domaines : si des analyses infra-observatoire (par commune, fokontany ou hameau) sont requises, une modélisation de type Battese-Harter-Fuller ou Fay-Herriot est préférable à une estimation directe, compte tenu des effectifs réduits.

  4. Prudence dans les comparaisons temporelles : la rupture du panel (2015–2025) et le renouvellement complet de l’échantillon empêchent de distinguer les évolutions démographiques réelles des effets de composition de l’échantillon. Les analyses longitudinales doivent signaler cette limite.
